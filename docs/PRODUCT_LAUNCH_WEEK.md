# Product Launch Week

## Objetivo
Sacar al mundo el sandbox de audio esta semana como producto funcional, sin mezclar esta fase con el core canónico del paper.

## Base técnica
- Rama base estable: `main`
- Hito estable: `hito-nivel1-audio-stable`
- Commit estable: `ca632e1`

## Regla
Esta semana solo se permiten tareas que acerquen el producto a publicación y validación externa.

## Qué sí hacer
1. README orientado a usuario
2. Demo pública clara
3. Tres audios de ejemplo buenos
4. Validación final de Streamlit
5. Texto corto de propuesta de valor
6. Material de salida: GitHub + portafolio + post

## Qué no hacer
- no implementar el operador canónico del paper
- no refactor profundo
- no benchmarks científicos
- no deuda técnica no bloqueante
- no features nuevas por ansiedad

## Checklist
### A. Producto
- [ ] nombre final visible del producto
- [ ] descripción corta
- [ ] instrucciones de uso simples
- [ ] parámetros entendibles
- [ ] export WAV validado

### B. Repositorio
- [ ] README limpio
- [ ] screenshots o GIF
- [ ] ejemplos de audio documentados
- [ ] CHANGELOG actualizado

### C. Demo
- [ ] Streamlit operativo
- [ ] prueba desktop
- [ ] prueba móvil
- [ ] prueba upload + process + download

### D. Salida pública
- [ ] GitHub público limpio
- [ ] blendedpareto.com actualizado
- [ ] texto de lanzamiento
- [ ] primer post preparado

## Criterio de cierre
Producto públicamente visible, usable y compartible.
